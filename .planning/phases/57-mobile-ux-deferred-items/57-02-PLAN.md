    File content shape (using Svelte 5 runes; the `.svelte.ts` extension is mandatory per CONVENTIONS.md "File Naming for Rune-based Modules"):
    ```ts
    const BREAKPOINTS = { sm: 640, md: 768, lg: 1024, xl: 1280 } as const;
    export type BreakpointName = keyof typeof BREAKPOINTS;

    export function useBreakpoint(name: BreakpointName): { readonly below: boolean } {
      const px = BREAKPOINTS[name];
      let below = $state(
        typeof window !== "undefined" ? window.innerWidth < px : false,
      );
      if (typeof window !== "undefined") {
        const mql = window.matchMedia(`(max-width: ${px - 1}px)`);
        const onChange = () => { below = mql.matches; };
        mql.addEventListener("change", onChange);
        $effect.root(() => () => mql.removeEventListener("change", onChange));
      }
      return { get below() { return below; } };
    }
    ```

    Critical decisions to lock per RESEARCH ¬ßPattern 1 + W0 Task 1 test contract:
    - SSR path returns a frozen `{ below: false }` (the matchMedia branch is skipped under SSR).
    - `addEventListener('change', ...)` over deprecated `addListener` ‚ÄĒ `matchMedia.addListener` was deprecated in Safari 14+.
    - `$effect.root(() => () => cleanup)` is the correct rune pattern for module-level effects. The function returns the getter object IMMEDIATELY ‚ÄĒ cleanup is registered as a side-effect.
    - Pixel thresholds match Tailwind defaults exactly (sm=640, md=768, lg=1024, xl=1280) ‚ÄĒ Wave 0 test 4 (`use-breakpoint.unit.test.ts`) asserts these literals.

    Do NOT add a `useBreakpointAbove` variant ‚ÄĒ only the `below` API is needed for UX-01 (the 9 pickers all wrap on `<lg`). YAGNI per CLAUDE.md DRY rule.

    <script lang="ts">
      import type { Snippet } from "svelte";

        open: boolean;
        onclose: () => void;
        ariaLabel?: string;

      let visible = $state(false);

          requestAnimationFrame(() => { entering = true; });

          const t = setTimeout(() => { visible = false; }, 300);
          return () => clearTimeout(t);

          const onKey = (e: KeyboardEvent) => {

          return () => { cleanup(); window.removeEventListener("keydown", onKey); };
        }
      });

        role="dialog" aria-modal="true" aria-label={ariaLabel}
