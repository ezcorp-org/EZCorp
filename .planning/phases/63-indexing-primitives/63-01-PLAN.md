  <action>
    Create `src/memory/message-chunker.ts` as a PURE-logic module (no I/O, no DB, no
    embeddings import beyond the type). Export:

    ```typescript
    import type { PreTrainedTokenizer } from "@huggingface/transformers";

    export const CHUNK_TOKENS = 256;
    export const OVERLAP_TOKENS = 32;

    export const EMBED_ELIGIBLE_ROLES = new Set(["user", "assistant"]);

    export function isEmbedEligible(role: string, content: string): boolean {
      return EMBED_ELIGIBLE_ROLES.has(role) && content.trim().length > 0;
    }

    export function chunkByTokens(tokenizer: PreTrainedTokenizer, text: string): string[] {
      const ids = tokenizer.encode(text, { add_special_tokens: false });
      if (ids.length <= CHUNK_TOKENS) return [text];
      const chunks: string[] = [];
      const stride = CHUNK_TOKENS - OVERLAP_TOKENS; // 224
      for (let i = 0; i < ids.length; i += stride) {
        const window = ids.slice(i, i + CHUNK_TOKENS);
        chunks.push(tokenizer.decode(window, { skip_special_tokens: true }));
        if (i + CHUNK_TOKENS >= ids.length) break;
      }
      return chunks;
    }
    ```

    PITFALL (research Pitfall 2): use `add_special_tokens: false` on encode so CLS/SEP
    do not consume the 256 budget; the extractor adds them back at embed time. Decode
    with `skip_special_tokens: true`.

    Write `src/__tests__/message-chunker.test.ts` covering every behavior above with a
    fake tokenizer (split on whitespace for encode = `text.split(/\s+/).map(...index)`,
    join for decode). For the overlap assertion, build a fake where token ids map 1:1 to
    word positions so you can assert window boundaries deterministically.

    DRY: do NOT duplicate the char-based logic in `src/memory/chunking.ts` — this is the
    token-aware replacement for chat (chunking.ts stays for KB file ingestion).
  </action>
