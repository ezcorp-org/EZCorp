### Integration gap (low severity, 1) — ✅ RESOLVED (commit `4bbd279b`)

**UI-03 / PAL-07 — chat-index deep-link drop**
`web/src/routes/(app)/project/[id]/chat/+page.svelte:57` — `handleSelect(id: string)` omitted the optional `messageId?: string` second arg that `ConversationList.onselect`/palette emit, so the `?m=` param was never appended and `ChatThread` never scrolled/pulsed. Type-safe (extra arg ignored), so no svelte-check error caught it.
**Blast radius:** only when a message-search hit is clicked from the empty chat-index page (before any conversation is selected) — primarily the mobile landing view.
**Fix applied:** `handleSelect` now forwards `messageId`, mirroring the `[convId]` sibling route's URL shape, and a chat-index component regression test was added (`web/src/__tests__/chat-index-select-deeplink.component.test.ts`, 2 cases GREEN) — the existing `sidebar-search-deeplink.spec.ts` e2e only exercised the `[convId]` route, which is why the index route was uncovered.
