### Integration gap (low severity, 1)

**UI-03 / PAL-07 — chat-index deep-link drop**
`web/src/routes/(app)/project/[id]/chat/+page.svelte:57` — `handleSelect(id: string)` omits the optional `messageId?: string` second arg that `ConversationList.onselect`/palette emit, so the `?m=` param is never appended and `ChatThread` never scrolls/pulses. Type-safe (extra arg ignored), so no svelte-check error caught it.
**Blast radius:** only when a message-search hit is clicked from the empty chat-index page (before any conversation is selected). Desktop usually redirects past this view; on mobile it is the primary list view, so the deep-link scroll/pulse is silently lost there.
**Fix (1 line):**
```ts
function handleSelect(id: string, messageId?: string) {
  goto(messageId
    ? `/project/${projectId}/chat/${id}?m=${encodeURIComponent(messageId)}`
    : `/project/${projectId}/chat/${id}`);
}
```
